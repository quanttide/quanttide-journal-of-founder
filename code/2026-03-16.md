# 2026-03-16

问了四五个 AI，大家的说法各自不一，总体来说提到的原则都是老生常谈的方法，感觉还不是最重要的编程实践原则。就是和当前 Agent 能力相关的最佳实践。从目前日志记录的情况，以及现场的情况来看，最主要的工作还是出现在没有被设计覆盖到的项目整体，这其实很好的说明了覆盖详细的设计对于工程质量控制的必要性。足够准确的文档是整个仓库的主要事实来源。以文档为中心而不在是以代码为中心。这是一个很重要的变化。

我的活难做主要是因为他们大多数具备很强的探索性工作，同时我对于代码的整洁度有非常高的洁癖，这让不擅长控制细节的 AI 学习是比较困难的。所以采取立宪的思想或许是一个不错的办法。

默认编程日记还是继续保持这种随性的风格，让元认知可以更自然而然地涌现，从而为正式开发理清道路。

具体项目的编程日志或许也可以这样随性。我觉得只 要让想法跑出来就可以了。至于其他的先不管它，以后再说。

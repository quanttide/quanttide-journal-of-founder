## 交付格式

这是一个非常符合“被动模式”的输出设计。我们定义一种“嵌入式批注格式”。
这种格式遵循一个原则：绝对不改动用户的原始文本，而是在逻辑段落之间插入“观察者视角”的结构化信息。 这样既保留了原始的“事件流”和“思维链”，又叠加了 AI 提炼的“语义层”。
设计规范：嵌入式批注格式
1. 插入位置： 在每个相对独立的思维段落（或“自然段”）之后。
2. 标识符号： 使用 Markdown 的引用语法 >，搭配特定的 Emoji 前缀，以便人眼快速区分。
3. 批注结构：
🏷️ 标签：对内容进行定性（如 #核心洞察、#待办、#工程决策）。
💎 提炼：从口语中提取的标准化知识或命题。
🔗 关联：识别到的潜在关联或冲突。

## 处理逻辑

基于你认可的“嵌入式批注”格式，以及之前确定的“被动响应式”理念，我为你设计了一个“增量式被动观察流程”。
这个流程的核心在于：AI 不负责全篇重写，只负责“扫描”和“打标”，就像一个在图书馆里整理卡片的助手，不打扰你的写作，只在你停顿（触发命令）时工作。
流程设计：增量观察者协议
第一阶段：输入与识别
触发机制：
当前模式：你在终端执行 python agent.py design-log.md。
未来模式：watchdog 监听到文件变更，且超过 30 秒无新写入（表示你思考暂停）。
文本切片：
脚本读取文件，按 \n\n（空行）将全文切分为多个 “逻辑段落”。
过滤：忽略只有标点、代码块或过短（<20字）的段落。
状态检测：
脚本检查每个段落末尾是否已存在 > **🤖 观察者注** 标记。
已存在 -> 跳过（保持幂等性，不重复消耗 Token）。
不存在 -> 标记为 “待观察段落”。
第二阶段：思考与提炼
针对每个“待观察段落”，AI 进行一次轻量级推理。
Prompt 策略：
“你是一个安静的观察者。阅读以下段落，判断其核心价值。
如果它只是无意义的闲聊或过渡语，请输出：SKIP。
如果它包含洞察、决策或风险，请按以下格式输出批注：
> **🤖 观察者注** ...”
处理策略：
降噪：对于“嗯、啊、那个”等口语，AI 在提炼时会自动忽略，但绝不修改原文。
提取：识别出“知识卡片”、“决策点”、“待办事项”。
第三阶段：合成与回写
这是最关键的一步，保证格式整洁。
合并输出：
将原文段落 + AI 生成的批注，拼接成一个新的文本块。
例如：[原始段落内容]
> **🤖 观察者注**
> ...
文件更新：
使用 Python 的字符串替换，将原文中的该段落替换为“拼接后的新块”。
写回文件。
伪代码实现
为了让你更直观地理解，这是简化的逻辑代码：
def process_journal(file_path):
    # 1. 读取
    with open(file_path, ’r‘, encoding=’utf-8‘) as f:
        content = f.read()
    # 2. 切分段落
    paragraphs = content.split(’\n\n‘)
    new_content_parts = []
    for para in paragraphs:
        # 3. 检查是否已处理
        if is_already_annotated(para): # 检查是否包含特定标记
            new_content_parts.append(para)
            continue
        # 4. 判断是否值得处理 (比如字数太少)
        if len(para.strip()) < 20:
            new_content_parts.append(para)
            continue
        # 5. 调用 AI 获取批注
        print(f”正在观察段落: {para[:20]}...“)
        annotation = get_ai_annotation(para) # 调用 LLM
        # 6. 拼接
        if annotation != ”SKIP“:
            updated_para = f”{para}\n\n{annotation}“
            new_content_parts.append(updated_para)
        else:
            new_content_parts.append(para)
    # 7. 写回
    with open(file_path, ’w‘, encoding=’utf-8‘) as f:
        f.write(’\n\n‘.join(new_content_parts))
    print(”观察完成。“)
def is_already_annotated(text):
    # 简单的标记检查
    return ”**🤖 观察者注**“ in text
这个流程的优势
增量处理：
你写一段，它处理一段。你改了以前的一段，它会重新处理那一段（如果它检测到标记丢失或不匹配）。
成本控制：随着文档变长，你只需要为新写的段落付费，不需要每次都重跑全文。
非侵入性：
你的原始文字（第一行到最后一行）一个字都不会动。
AI 的输出永远在“缝隙”中（段落末尾的引用块）。
这完美解决了你担心的“AI 瞎改原文”的问题。
人机对齐：
当你看到 AI 标注了 #待办 但你其实已经做完了，你会直观地看到这个差异，并在心里（或实际操作中）修正它。
这就是你在日记里说的“人去学习到了我的叙事流”的双向过程。
